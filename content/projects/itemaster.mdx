Designed for businesses of all sizes, this intuitive platform helps you track stock levels, automate reordering, and generate detailed reports—all in real-time. Accessible from any device, it simplifies inventory management, reduces errors, and boosts efficiency. Say goodbye to manual tracking and hello to seamless operations.

## Features

- **Item nanagement**: Manage and categorize inventory items.
- **Unit of measurement conversions**: Track stocks using different unit of measurements.
- **Inventory management**: Manage item inventory in each different location.
- **Point of sales**: Simple POS for inputing sales.
- **Reports**: Generate reports of purchases, sales, and inventory.
- **Movement history**: Realtime tracking of remaining stocks.
- **Statistics**: Real-time visual representation of sales and purchases.

## Tech Stack

- **Frontend**: Bootstrap and JQuery for building dynamic and responsive user interfaces.
- **Backend**: Laravel for handling server-side logic.
- **Database**: MySQL for managing user data and inventory records.
- **Hosting**: Deployed on a cloud platform with CI/CD for seamless updates.

## Implementation

### Frontend

- Built using **Bootstrap and JQuery**.
- Interactive elements like graph implemented using libraries like **ChartJS**.

### Backend

- Developed with Laravel.
- Authentication implemented with **Built-in Laravel Authentication** for secure user sessions.

### Database

- **MySQL** used for storing user details, inventory records, purchases, and sales.
- Optimized queries to ensure smooth and fast data retrieval.

### Deployment

- Hosted on **AWS EC2**

## How It Works

1. **Create Items**: Create and categorize items.
2. **Update Stocks**: Update item stocks using Stock Adjustment or Purchase Order.
3. **Create Sales**: Input Sales on POS.
4. **Generate Report**: Generate purchases, stock adjustments, sales, and stock report.

## Challenges Faced

- Ensuring accurate computation.
- Designing an intuitive UI that works seamlessly across devices.
- Handling performance as the database grows.
